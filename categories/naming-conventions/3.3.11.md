# 3.3.11 - Call Options Options

Priority | ID | Category | Criteria | Impacts
---------|----|----------|----------|--------
Minor | [3.3.11][3.3.11] | Naming Conventions | All user-supplied option classes/interface names must end with `Options`. | [Adoption][ADOPTION], [Contributions][CONTRIBUTIONS]

Table of Contents:
- [Description](#description)
- [Discussion](#discussion)

## Description

All user-supplied option classes/interface names must end with `Options`.

This rule applies to any class or interface that can be named as the [user-supplied options][User-Supplied Options] parameter to a function.

Do not use the `Options` suffix for another kind of class or interface.

```typescript
export interface OnErrorOptions {
    onError?: OnError;
}

function doSomething(
    input: string,
    { onError = THROW_THE_ERROR }: OnErrorOptions = {}
) {
    ...;
}
```

## Discussion

This is the common style used in the Typescript world.
