# 3.3.9 - Private Class Attributes Start With An Underscore

Priority | ID | Category | Criteria | Impacts
---------|----|----------|----------|--------
Minor | [3.3.9][3.3.9] | Naming Conventions | Prefix private and protected class attributes with an underscore. | [Adoption][ADOPTION], [Contributions][CONTRIBUTIONS]

Table of Contents:
- [Description](#description)
- [Discussion](#discussion)

## Description

Prefix private and protected class attributes with an underscore.

```typescript
export class ValueObject {
    // value is a class property
    private _value: string;
}
```

This does not apply to ECMA private fields:

```typescript
export class ValueObject {
    // #value is an ECMA private field
    #value: string;
}
```

## Discussion

This is the common style used in the Typescript world.

It also creates the space needed to write a getter:

```typescript
export class ValueObject {
    protected _value: string;

    // this would clash with the attribute, if they
    // had the same name!
    get value(): string {
        return this._value;
    }
}
```
