# 3.3.13 - Name All DEFAULT_OPTIONS Consistently

Priority | ID | Category | Criteria | Impacts
---------|----|----------|----------|--------

## Description

All default options must be called `XXX_DEFAULT_OPTIONS`.

_Default options_ are default values for [user-supplied options][User-Supplied Options]. They're available for the [caller][Caller] to pass in. See [2.8.10][2.8.10] for details.

For example:

```typescript
// if we have a function called isType() ...
export function isType(...);

// its default options will be called:
export const IS_TYPE_DEFAULT_OPTIONS = { ... };
```

## Discussion

### Make Things Easy To Discover

We give all _default options_ a predictable name so that [end-users][End-User] can easily discover them in their IDE. All they've got to do is learn our convention.

### Make Sure Names Do Not Clash

We follow the same naming scheme for all _default options_ so that there's always the space to create more of them in the future.
