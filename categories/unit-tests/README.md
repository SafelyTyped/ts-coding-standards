## Introduction

The _Safe Types movement_ is, at heart, an initiative to improve software quality. We've chosen to do that by moving [defensive programming][Defensive Programming] from literally everywhere (or nowhere, for projects that don't do any defensive programming) into [type constructors][Smart Constructor].

_Unit tests_ are just as importing to our goal as the safe types themselves.

* We aim to ship code, not bugs.
* We aim to ship new features, not new regressions.
* We aim to maintain backwards-compatibility, not break it.

Unit testing is one of our major tools for achieving those aims.

## What Is In-Scope?

Currently in-scope is:

* the existence of unit tests
* how we write unit tests

## What Is Out Of Scope?

Currently out-of-scope includes:

* how to write code that's easier to test
