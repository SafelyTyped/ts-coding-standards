## Description

Put exported default values in their own file.

```typescript
// src/v1/Retrieve/defaults/DEFAULT_CONTENT_TYPE.ts
export DEFAULT_CONTENT_TYPE="application/octect-stream";
```

Although it isn't a coding standard, you might prefer to keep these files in a `defaults/` sub-folder.

## Discussion

### Make Source Code Easy To Find

We are putting exported default values into their own files to make it easier to navigate around a source tree.

Most modern IDEs and advanced text editors support a "Goto File" feature. We can type a partial filename, and the editor will show us a list of matching files to choose from. It's a really handy way to navigate quickly around a source tree - especially if it's one you don't know very well.

### Make Changes Easier To Find

We are putting exported default values into their own files to make code reviews easier.

This coding standard helps project maintainers see what code has been modified at a glance. They can look at a pull request, and use the list of modified files to determine what code has been modified.
